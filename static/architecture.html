Architecture Overview

How the tool processes project documents through a four-stage LLM-assisted FCV analysis pipeline to produce a Recommendations Note for Task Team Leaders.

Project Document

Required
Contextual Documents

Optional

4 · Human Oversight — Where the user intervenes
Refine at any stage
After each stage, the user can read the AI's reasoning and submit free-text instructions to correct, redirect, or expand the analysis before proceeding. The AI re-runs the same stage with the new instruction.
Edit stage prompts
Users can view and edit the prompt instructions sent to the AI at each stage via the built-in prompt editor. Edits apply to the current session only — defaults reload on next visit. Modified prompts are clearly flagged.
Save and resume sessions
Users can save their session as a JSON file at any point and reload it later to continue from where they left off. Conversation history across all completed stages is preserved.
Copy output to Word
Each stage output can be copied to clipboard in one click. The Stage 3 Recommendations Note includes a mandatory AI disclaimer prepended automatically before any content.
5 · Output
FCV Recommendations Note for the Task Team Leader
A structured peer-review memo (~3,500 words) with: an executive summary and opening assessment; identified strengths and gaps; operational context; 4–5 strategic priorities for preparation; and a technical annex with thematic risk pillars and options to consider. Produced as streaming text, copyable to Word.
