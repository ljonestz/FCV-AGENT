# WB LLM Review Prompt — FCV Project Screener v8: Output Quality Review

## Context

I am developing an AI-assisted tool to help World Bank Task Team Leaders (TTLs) assess the FCV (Fragility, Conflict and Violence) sensitivity and responsiveness of project documents. The tool runs a three-stage analysis pipeline:

- **Stage 1:** Extracts project context, instrument type, temporal anchoring, and PDO/scope from uploaded documents, then contextualises with web research
- **Stage 2:** Assesses the project's FCV integration using the WBG OST framework, FCV Refresh shifts, and Do No Harm principles — generating ratings for FCV Sensitivity and Responsiveness
- **Stage 3:** Produces prioritised, actionable recommendations with specific language for inclusion in project documents

I have shared the following documents with you:
1. **01_Knowledge_Base_v8.docx** — what the tool knows about WB instruments (IPF, PforR, DPO, TA, MPA, IPF-DDO), implementation processes, and FCV terminology
2. **02_Stage_Prompts_v8.docx** — the exact instructions given to the AI at each stage
3. **03_FCV_Reference_Constants_v8.docx** — the FCV framework content the tool uses (FCV Strategy, Operational Manual, Refresh Framework, Playbooks)
4. **stage 1 output.docx** — actual output generated by the tool for a real project (Honduras Sustainable Connectivity Project, P181166, $187M IPF, November 2024)
5. **stage 2 output.docx** — the FCV assessment produced for that project
6. **stage 3 output.docx** — the recommendations note produced for that project

I am asking for your expert review as an FCV practitioner and WBG insider. Please assess whether the tool's outputs are **sufficient, accurate, and well-structured** based on WBG internal FCV guidance and your understanding of what a high-quality FCV sensitivity and responsiveness assessment should look like.

---

## Review Questions

### 1. Stage 1 — Context and Extraction Quality

Please review **stage 1 output.docx** and assess:

a) Has the tool correctly identified the instrument type (IPF), document type (PAD), and project stage?

b) Is the temporal anchoring accurate — does it correctly identify the approval date, safeguards framework (ESF vs. old OP/BPs), and assess the project by the standards available at preparation?

c) Is the PDO, Theory of Change, and Results Framework scope accurately captured? Are there any material omissions?

d) Does Part A (document-sourced content) stay within the bounds of what the uploaded PAD says? Does Part B (contextualised with external knowledge) add genuine analytical value, or introduce noise/hallucination?

e) Are there any factual errors, mischaracterisations, or missed content that would affect downstream analysis?

---

### 2. Stage 2 — FCV Assessment Quality

Please review **stage 2 output.docx** and assess:

a) **FCV Refresh framing:** Does the assessment correctly apply the four FCV Strategy Refresh shifts (Anticipate, Differentiate, Jobs & Private Sector, Enhanced Toolkit)? Are they applied accurately to the project's actual content, or mechanically?

b) **Sensitivity vs. Responsiveness:** Are the [S], [R], and [S+R] tags correctly applied? Is the distinction between FCV sensitivity (conflict-informed design, Do No Harm) and FCV responsiveness (actively working to change FCV conditions) correctly maintained throughout?

c) **Instrument appropriateness:** Does the assessment correctly apply IPF-specific criteria? Does it avoid penalising the project for things that are structurally outside an IPF's scope (e.g., policy conditionality, DLIs, DPO-style reform)?

d) **Do No Harm assessment:** Is the traffic-light (addressed / partial / gap) for the 8 Do No Harm principles accurate and grounded in the project content?

e) **Rating calibration:** Are the Sensitivity and Responsiveness ratings (scale: Very Low / Low / Adequate / Good / Strong / Excellent) appropriately calibrated for this project type and FCV context? If you would rate this project differently, please explain why and what the rating should be.

f) **Logical consistency:** Are there any contradictions, unsupported claims, or analytical tensions that should be explicitly reconciled?

g) **Missing dimensions:** Are there FCV considerations material to this project that the assessment missed entirely?

---

### 3. Stage 3 — Recommendations Quality

Please review **stage 3 output.docx** and assess:

a) **Specificity:** Are the priority recommendations specific enough to act on — do they name geographic areas, implementation mechanisms, and concrete entry points? Or are they too generic?

b) **Instrument feasibility:** Are all recommendations feasible under an IPF? Identify any that would require DPO-style conditionality, PforR-style DLIs, or other instruments not available to this team.

c) **Grounding in evidence:** Are the recommendations grounded in the uploaded project document, or do they appear to be generic FCV advice not specific to Honduras or this sector?

d) **Horizon Considerations:** Is the Horizon Considerations section (items beyond current scope) appropriately populated? Does it correctly separate what is addressable within this operation from what requires a future operation or policy dialogue?

e) **Suggested language quality:** Is the "suggested language" provided in the priority actions ready to use as PAD text, or does it need significant editing? Would a TTL find this language helpful?

f) **PAD section labels:** Are the recommended PAD sections (Strategic Context, Project Description, Implementation Arrangements, etc.) correctly identified for where each recommendation should be embedded?

g) **Completeness:** Are there priority FCV actions for this project that the tool missed entirely?

---

### 4. Overall Sufficiency Assessment

a) Would a TTL or FCV Country Coordinator find these outputs **sufficient to inform real design decisions** on this project — or are there gaps that would require them to redo the analysis manually?

b) What is the **single most important improvement** that would meaningfully raise the quality of these outputs?

c) Are there any outputs that are **actively misleading** — i.e., where following the tool's advice would lead a TTL to make a worse decision than if they had not used the tool at all?

d) On a scale of **1–5** (1 = not useful, 3 = useful with caveats, 5 = ready to use as-is), how would you rate:
   - Stage 1 output quality: ___
   - Stage 2 assessment quality: ___
   - Stage 3 recommendations quality: ___

---

### 5. Knowledge Base and Prompt Accuracy (Optional but Welcome)

If you have time, please also flag any inaccuracies you notice in the reference documents:

a) **01_Knowledge_Base_v8.docx:** Are the instrument descriptions (IPF section in particular) accurate? Are there operational procedures, flexibilities, or guardrails that are missing or misstated?

b) **02_Stage_Prompts_v8.docx:** Are the instructions given to the AI likely to produce the kind of assessment a WBG FCV practitioner would recognise as rigorous and correct?

c) **03_FCV_Reference_Constants_v8.docx:** Is the FCV framework content (Refresh shifts, Playbook guidance) accurately represented and current?

---

## Format for Your Response

Please structure your response by section (1 through 4, with 5 optional). Within each section, be as specific as possible — reference the actual output text where relevant. Prioritise actionable feedback: what should be changed and why, not just what is wrong.

If you identify errors, please distinguish between:
- **Critical errors** (would lead to wrong decisions)
- **Material gaps** (important but not misleading)
- **Minor issues** (polish / refinement)

Thank you.
